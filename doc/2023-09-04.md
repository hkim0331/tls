### クオート

- リストはデータでもあり、
- リストはプログラムでもある。

データとプログラムを区別するために ' を使う。

- '(1 2 3) ... 三つのアトム 1,2,3 からなるデータ（リスト）。

- (1 2 3) ... 三つのアトム 1,2,3 からなるリストだが、リストの先頭要素の1を関数、
それ以外を関数の引数と解釈し、関数の実行を試みる。

クオートのつかないリストの先頭要素は関数なので、リストの先頭要素の場所を
`関数ポジション`
なんて呼ぶとかっこいいかな 😎

### 評価の順番

関数の引数を評価後に、関数が評価される。
文章で難しくても、ごく自然だから。

```scheme
(f (g a b) (h c d))
```

1. (g a b) が評価され、その評価値を G としよう。
2. (h c d) が評価され、その評価値を H としよう。
3. G, H がもとまったあと、(f G H) が評価される。


### car
car の代わりに first を使うのが流行り。リストの先頭要素。

```scheme
(first '(apple orange banana))
;=> 'apple
(first '((apple) (orange) (banana)))
;=> '(apple)
(first '(1 2 3))
;=> 1
```

### cdr
cdr の代わりに rest を使うのが流行り。リストから先頭要素を除いた残り。

```scheme
(rest '(apple orange banana))
;=> '(orange banana)
(rest '((apple) (orange) (banana)))
;=> '((orange) (banana))
(rest '(1 2 3))
;=> '(2 3)
```

### null?
一引数で呼ばれ、引数が空リストであれば #t を返し、
それ以外は #f.

```scheme
(null? 1)
;=> #f
(null? '())
;=> #t
(null? '(1 2 3))
;=> #f
```

### eq?
二引数で呼ばれ、引数が等しければ #t、
それ以外は #f を返す。引数はアトムに限る。

```scheme
(eq? 1 1)
;=> #t
(eq? 1 2)
;=> #f
(eq? '(1) '(1))
;=> #f
```


### cons
first と rest がリストを分解するのに対し、
cons はリストを合成する。

```scheme
(cons 1 '(2))
;=> '(1 2)
(cons '(1) '(2))
;=> '((1) 2)
(cons '1 '())
;=> '(1)
(cons '() '())
;=> '(())
```

car(first), cdr(rest), cons, null?, atom?
を組み合わせ、いろんな関数を作る。

## 第2章

### 関数 lat? を定義する

全要素がアトムであるリストをこの本では lat と呼んでいる。
List of AToms の頭文字だろうな。

リスト x が lat かどうかを判定する関数 lat? をプログラムしたい。
???? に入るのは何でしょう？

```
(define lat?
  (lambda (x)
    (cond
      ((null? x) #t)
      ((atom? (first x)) (???? (rest x)))
      (else #f))))
```

日本語で考えてみる。
- (null? x) が #t になる時、x は '()。lat だわな。空ラット。
- x の先頭要素がアトムなら、(lat? x) の真偽は (lat? (rest x)) に等しい。
- x の先頭要素がアトムじゃないなら、(rest x) をみるまでもなく、x は lat ではないと判断できる。
  例えば、 x が '((1) 2 3 4 5 6 7) のケース。

この本では lambda が何だとか説明なしに、こんな感じでズブっと出てくる。
読者は想像を豊かに、自分で考え、自分の定義らしきものを考えることが求められる。

cond は多分岐。scheme には if もあるけど、if は二分岐。

```scheme
(cond
  ((ここが #t になったら) これ)
  ((上が #f で,ここが #t になったら) これ)
  ((上が #f で,ここが #t になったら) これ)
  ((上が #f で,ここが #t になったら) これ)
  ...
  (else (上が全部 #f だったらこれ)))
```

```scheme
;; 参考: x の偶奇で分岐するif はこう書く。
;; tls では分岐はすべて cond でプログラムしている。if は出てこない。
(if (even? x)
  (println "x は偶数だね。")
  (println "x は偶数じゃありません。"))
```
